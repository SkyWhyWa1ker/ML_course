# SIS-3: ML System with Streamlit + MLflow

Extension of Practical Task 6 — adds a Streamlit frontend and MLflow experiment tracking + model registry.

## Project Structure

- `train.py` — trains model, logs to MLflow, registers in Model Registry
- `main.py` — FastAPI prediction API
- `streamlit_app.py` — Streamlit frontend UI
- `model.joblib` — saved model (generated by train.py)
- `requirements.txt` — all dependencies
- `Dockerfile` — FastAPI container
- `Dockerfile.streamlit` — Streamlit container
- `docker-compose.yml` — orchestrates all 3 services

## Services

| Service   | URL                    | Description              |
|-----------|------------------------|--------------------------|
| MLflow    | http://localhost:5000  | Experiment tracking UI   |
| FastAPI   | http://localhost:8000  | Prediction API + Swagger |
| Streamlit | http://localhost:8501  | User-facing frontend     |

## How to Run

### Step 1: Train the model (logs to MLflow)

Make sure MLflow is running first, then:

```bash
docker-compose up mlflow -d
python train.py
```

### Step 2: Start all services

```bash
docker-compose up --build
```

### Step 3: Open in browser

- Streamlit UI: http://localhost:8501
- MLflow dashboard: http://localhost:5000
- FastAPI Swagger: http://localhost:8000/docs

## MLflow Features

- **Experiment Tracking**: logs `n_estimators`, `max_depth`, `random_state`, `accuracy`, `f1_score`, and the model artifact
- **Model Registry**: model is registered as `IrisClassifier` with automatic versioning

## Example API Request

```json
POST /predict
{
  "sepal_length": 5.1,
  "sepal_width": 3.5,
  "petal_length": 1.4,
  "petal_width": 0.2
}
```

Response:
```json
{
  "prediction": 0,
  "class_name": "setosa"
}
```
